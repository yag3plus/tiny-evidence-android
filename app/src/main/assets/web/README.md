# Tiny Evidence 1.4

这是一次围绕“好看、可选、即时安抚”的大版本重构。

## 核心变化

- 左上角统一展开菜单
- 极简首页，不显示任务和统计
- 单输入框记录，心情与类型默认折叠
- 移除价值连接、身份标签、“它证明了什么”等内容
- “成长证据库”改为“我的痕迹”
- 保留可视化日历，但不显示连续天数、遗漏或完成率
- 日历支持完整月历、只看留下的日子、暂时隐藏
- 增加温和的数据可视化：记录类型与可选心情
- 增加独立的安静空间：
  - 呼吸动画
  - 自由倾倒
  - 简单着地练习
  - 安静停留
- 放松工具默认不写入日历；自由倾倒只有主动保存时才进入记录
- 随机探索改为用户主动请求的“小建议”，不产生未完成任务
- 可迁移旧版本浏览器中 `tiny-evidence-records-v1` 的本地记录

## 使用

直接打开 `index.html` 即可。

数据保存在浏览器 LocalStorage 中。清理浏览器数据会删除内容，请定期使用菜单里的“导出我的内容”。

## 以本地服务器运行

```bash
python -m http.server 8000
```

然后打开 `http://localhost:8000`。


## 1.1 新增：隐藏成长点宠物系统

- 首页增加可点击互动的小企鹅伙伴
- 支持自定义宠物名字
- 后台保存好奇、创造、活力、安静、连接五种隐藏属性
- 前台不显示数值、进度条或下一次解锁条件
- 只通过动作、状态文字和房间物品展现变化
- 小建议保持随机且不显示类型
- 小建议新增“已完成”按钮
- 完成后触发宠物动画、隐藏成长变化和日历记录
- 未完成、换一个或关闭页面不会留下失败记录
- 宠物不会饥饿、生病、倒退或责怪用户


## 1.2 新增（保留 1.1 原有 UI 与功能）

- 完整保留原来的小企鹅造型、房间、动画、首页、日历、安静空间、小建议和深色模式
- 新增“想告诉你一件事”
- 新增“待处理的事情”：重要性、截止时间、状态、备注和排序
- 新增可选伯恩斯反拖延症表
- 对照后弹出 Emoji 感受评分
- 1–2 分时可选输入想说的话
- 删除“这个感受也可以被留下”等文案
- 新增完整备份导出、合并导入和替换导入

## 1.4 早晚陪伴
- 新增“今天可以结束了”页面。

- 早晨拉开窗帘并留下太阳痕迹。
- 晚上关灯、盖毯子并留下月亮痕迹。
- 新增早晚月历；空白不代表失败。
- 网页版不包含定时消息；安卓版本额外包含本地通知。
